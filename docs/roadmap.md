# RepoMind Roadmap

## 最终目标

RepoMind 是一个面向开发者和编程学习者的开源项目学习与二次开发 Agent。

它的完整路径是：

GitHub 项目理解  
→ 项目导读  
→ 项目问答  
→ 多轮追问  
→ 检索自评  
→ 运行辅助  
→ 二次开发建议  
→ patch 生成  
→ Debug 与验证

最终希望 RepoMind 成为一个能够帮助用户理解项目、学习项目、运行项目，并辅助二次开发的开发协作 Agent。

---

## V0.1 基础数据链路

目标：让 RepoMind 能拿到一个 GitHub 项目，并生成基础项目材料。

路径：

GitHub URL  
→ 克隆仓库  
→ 扫描项目目录  
→ 过滤无关文件  
→ 读取关键文件  
→ 规则分析项目类型、技术栈、入口文件、核心目录  
→ 生成 Markdown 基础报告

---

## V0.2 大模型项目导读

目标：让 RepoMind 能基于真实项目材料生成易读的项目导读。

路径：

文件树  
→ 规则分析结果  
→ 关键文件内容  
→ 组织 Prompt  
→ 调用大模型  
→ 生成项目导读报告

---

## V0.3 项目问答雏形

目标：让用户可以围绕当前项目继续提问。

路径：

用户问题  
→ 判断是否需要检索源码  
→ 如果需要，生成搜索计划  
→ 使用 ripgrep 检索代码  
→ 扩展相关代码上下文  
→ 大模型基于上下文回答问题

---

## V0.4 Web 交互雏形

目标：让 RepoMind 从终端脚本变成可交互的小工具。

路径：

输入 GitHub URL  
→ 点击分析项目  
→ 展示项目导读  
→ 输入问题  
→ 展示 RepoMind 回答

---

## V0.5 多轮对话记忆

目标：让 RepoMind 能支持自然追问。

路径：

保存用户问题和回答  
→ 记录最近几轮对话  
→ 下一轮提问时带上历史上下文  
→ 支持“它”“这个设计”“刚才那个模块”这类追问

---

## V0.6 检索自评与二次检索

目标：让 RepoMind 能判断当前信息是否足够。

路径：

第一次检索  
→ LLM 判断检索结果是否足够  
→ 如果不够，生成第二轮搜索计划  
→ 再次检索  
→ 合并上下文  
→ 生成更可靠的回答

---

## V1.0 项目理解 Agent

目标：形成稳定的项目理解与学习辅助能力。

路径：

项目导读  
→ 项目问答  
→ 技术解释  
→ 学习建议  
→ 简化版复刻建议  
→ 多轮追问  
→ 基于项目上下文持续交流

---

## V1.5 运行辅助

目标：帮助用户理解并尝试运行项目。

路径：

识别 README  
→ 识别依赖文件  
→ 判断启动入口  
→ 生成运行步骤  
→ 辅助安装依赖  
→ 运行命令  
→ 读取输出和报错  
→ 给出排查建议

---

## V2.0 二次开发助手

目标：根据用户需求辅助分析修改方案。

路径：

用户提出二次开发需求  
→ 分析需求意图  
→ 判断影响范围  
→ 定位相关模块  
→ 生成修改计划  
→ 给出实现建议  
→ 生成可审查 patch

---

## V3.0 Debug / Patch Agent

目标：让 RepoMind 具备执行型开发辅助能力。

路径：

用户确认修改  
→ 应用 patch  
→ 运行测试或启动项目  
→ 读取报错日志  
→ 分析错误原因  
→ 生成修复方案  
→ 再次验证  
→ 必要时回滚修改

---

## 总体路线

V0.1 基础数据链路  
→ V0.2 大模型项目导读  
→ V0.3 项目问答雏形  
→ V0.4 Web 交互雏形  
→ V0.5 多轮对话记忆  
→ V0.6 检索自评与二次检索  
→ V1.0 项目理解 Agent  
→ V1.5 运行辅助  
→ V2.0 二次开发助手  
→ V3.0 Debug / Patch Agent
