# RepoMind 实现思路

RepoMind 是一个面向开发者和编程学习者的开源项目理解与二次开发 Agent。

它的目标不是简单总结 GitHub 项目，而是帮助用户把一个陌生开源项目转化为可理解、可提问、可运行、可复刻、可改造的项目资产。

核心链路是：

GitHub URL → 克隆仓库 → 扫描项目 → 生成理解报告 → 支持代码问答 → 辅助二次开发

---

## 用户视角

用户输入一个 GitHub 项目 URL 后，RepoMind 会生成一份易读的项目解读报告，包括项目定位、核心功能、技术栈、目录结构、入口文件、核心模块和学习建议。
d
随后，用户可以像和熟悉代码库的同事讨论一样，对项目进行多轮提问。RepoMind 会基于真实代码文件回答问题，帮助用户理解功能实现、模块关系和修改入口。

对于结构清晰的小型项目，RepoMind 可以尝试识别运行方式并辅助本地运行；对于复杂项目，则生成运行步骤、环境要求和报错排查建议。

在二次开发阶段，用户可以提出修改需求或复刻目标。RepoMind 会基于真实项目上下文定位相关文件，生成修改计划、影响范围说明和可审查的 patch。后期可以在用户确认后应用修改，并结合运行日志进行 Debug 辅助。

---

## 流程视角

RepoMind 首先接收 GitHub URL，并将仓库克隆到本地工作区。

克隆完成后，系统会扫描项目目录，过滤 `.git`、缓存文件、依赖包、图片、构建产物等无关内容，只保留对理解项目有价值的文件。

随后，系统会读取 README、依赖文件、配置文件、入口文件和核心源码，并识别项目技术栈、目录结构、核心模块和大致运行方式。

这些信息会被整理成结构化上下文，用于生成项目理解报告和新手阅读路线。

当用户继续提问时，RepoMind 不只依赖之前的总结，而是根据问题重新检索仓库中的相关文件，读取代码片段，再结合项目上下文生成回答。

在二次开发阶段，RepoMind 会根据用户需求检索相关模块，判断修改范围，生成修改计划和 patch。前期只生成可审查方案，后期再支持用户确认后应用修改、运行项目、读取日志并辅助 Debug。

---

## 模块视角

RepoMind 可以拆成以下核心模块：

- `repo_loader`：负责接收 GitHub URL，并克隆仓库到本地工作区。
- `file_scanner`：负责扫描目录、过滤无关文件、生成文件树。
- `file_reader`：负责读取 README、依赖文件、配置文件、入口文件和核心源码。
- `repo_analyzer`：负责识别项目类型、技术栈、核心模块、入口文件和运行方式。
- `report_generator`：负责生成项目理解报告和阅读路线。
- `code_searcher`：负责根据用户问题检索相关代码文件。
- `qa_agent`：负责基于代码上下文回答用户问题。
- `dev_assistant`：负责根据用户二次开发需求生成修改计划和 patch。

后期扩展模块：

- `agent_orchestrator`：负责判断用户意图，并调度不同模块。
- `patch_manager`：负责展示、应用和回滚 patch。
- `runner`：负责运行安装、启动、测试等命令。
- `debug_agent`：负责读取报错日志并生成修复建议。
